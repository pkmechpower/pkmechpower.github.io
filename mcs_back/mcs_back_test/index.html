    MCS_Back_Test Class

    Overview

    MCS_Back_Test is a specialized integration test class for the MCS_Back (Motor Control System - Back) board. It extends RTOS_Test with board-specific functionality including GPIO mocking, ADC simulation, motor driver interfaces, and state machine testing capabilities.

    Key Features

    • State Machine Testing: Direct access to MSM states and status flags

    • GPIO Simulation: Mock GPIO interfaces for all board signals

    • ADC Input Control: Set and verify analog input values

    • Motor Driver Permits: Monitor and test RFE/FRG permit signals

    • Application Data Access: Thread-safe reading of application state

    • State Initialization: Helper methods to quickly reach specific FSM states

    • Periodic ADC Updates: Automatic ADC interrupt generation

    SetUp()

    1
    virtual void SetUp() override;

    Performs comprehensive initialization:

    1. Creates network manager buffers and tasks

    2. Sets up process inputs task with ADC and wheel speed queues

    3. Configures water pump task

    4. Creates application task

    5. Registers ADC DMA interrupt handler

    6. Sets initial power supply voltage (12.0V)

    7. Triggers initial ADC conversion

    Important: Always call after any derived class setup:

    initialize_to_state()

    1
    void initialize_to_state(MSM_States state, bool is_prerequisite = false);

    Performs all necessary steps to reach a specific state, including sending required CAN messages and waiting for conditions.

    Parameters:

    • state: Target state

    • is_prerequisite: If true, suppresses "Initialized to" message (for chained initialization)

    Supported States:

    • INIT - Does nothing (initial state)

    • TS_READY_FOR_ACTIVATION - Waits for initialization

    • PRE_CHARGE - Sends TS activation request

    • ROTATING_FIELD_ENABLED - Sends precharge complete message

    • READY_TO_DRIVE - Waits for FRG permit delay and sends R2D activation

    • FAULTY - Not yet implemented

    Example: 

    1
2
3
4
5
    // Initialize to R2D state
initialize_to_state(MSM_States::READY_TO_DRIVE);

// Now in R2D, can test R2D-specific behavior
EXPECT_TRUE(get_gpio(GPIOs::MD_L_FRG_PERMIT).read());

    class PermitsTest : public MCS_Back_Test, 
                    public ::testing::WithParamInterface<MSM_States> {
};

INSTANTIATE_TEST_SUITE_P(
    , PermitsTest,
    ::testing::Values(
        MSM_States::INIT,
        MSM_States::TS_READY_FOR_ACTIVATION,
        MSM_States::PRE_CHARGE,
        MSM_States::ROTATING_FIELD_ENABLED,
        MSM_States::READY_TO_DRIVE
    )
);

TEST_P(PermitsTest, CorrectPermitStates) {
    auto target_state = GetParam();
    initialize_to_state(target_state);

    switch (target_state) {
    case MSM_States::READY_TO_DRIVE:
        EXPECT_TRUE(get_gpio(GPIOs::MD_L_RFE_PERMIT).read());
        EXPECT_TRUE(get_gpio(GPIOs::MD_L_FRG_PERMIT).read());
        break;
    case MSM_States::ROTATING_FIELD_ENABLED:
        EXPECT_TRUE(get_gpio(GPIOs::MD_L_RFE_PERMIT).read());
        EXPECT_FALSE(get_gpio(GPIOs::MD_L_FRG_PERMIT).read());
        break;
    default:
        EXPECT_FALSE(get_gpio(GPIOs::MD_L_RFE_PERMIT).read());
        EXPECT_FALSE(get_gpio(GPIOs::MD_L_FRG_PERMIT).read());
        break;
    }
}

    Important Notes

    Thread Safety

    Application and shared data access is thread-safe through mutex protection: 

    1
2
3
4
5
    // Safe - returns a copy
auto data = get_app_data();
auto shared = get_shared_data();

// Not safe - don't try to access internal structures directly

    ADC Periodic Updates

    ADC conversions are triggered automatically every 1ms via a periodic hook registered during SetUp(). Manual triggering is also possible:

    1
2
    set_adc_input(ADC_Inputs::POWER_SUPPLY_VOLTAGE, value);
trigger_isr(Simulator::Interrupt_Vector::MCS_BACK_ADC_DMA);  // Manual trigger

    State Initialization Overhead

    initialize_to_state() performs real initialization including:

    • Waiting for delays (e.g., 600ms FRG permit delay for R2D)

    • Sending required CAN messages

    • Verifying state transitions

    This means tests may take significant time when initializing to higher states.

    Best Practices

    1. Always initialize to the required state 

      1
2
3
4
5
6
      // GOOD
initialize_to_state(MSM_States::READY_TO_DRIVE);
// Test R2D-specific behavior

// BAD
// Manually try to replicate initialization

    2. Use condition-based waiting for state transitions 

      1
2
3
4
5
      // GOOD
simulate_until_in_state(MSM_States::READY_TO_DRIVE);

// BAD
simulate_for(1.0f);  // May be too short or too long

    3. Use helper functions for ADC values 

      1
2
3
4
5
6
      // GOOD
set_adc_input(ADC_Inputs::POWER_SUPPLY_VOLTAGE, 
              Simulator::helpers::voltage_to_reading(12.0f));

// BAD
set_adc_input(ADC_Inputs::POWER_SUPPLY_VOLTAGE, 2048);  // What does this mean?

    4. Check both state and status flags 

      1
2
3
      EXPECT_TRUE(is_in_state(MSM_States::READY_TO_DRIVE));
EXPECT_TRUE(get_status_flag(MSM_Status_Flags::is_ready_to_drive));
EXPECT_TRUE(get_status_flag(MSM_Status_Flags::are_frg_permits_granted));

    5. Allow time for processing 

      1
2
3
4
5
6
7
      // Send CAN message
receive_CAN_message(CAN_Instance::CAN_1, frame);

// Give time to process (at least 10ms for task cycle)
simulate_for(0.02f);

// Now check results

    Common Pitfalls

    Not Waiting After ADC Updates

    1
2
3
4
5
6
7
8
    // WRONG
set_adc_input(ADC_Inputs::POWER_SUPPLY_VOLTAGE, value);
auto data = get_shared_data();  // Old value!

// CORRECT
set_adc_input(ADC_Inputs::POWER_SUPPLY_VOLTAGE, value);
simulate_for(0.01f);  // Wait for processing
auto data = get_shared_data();  // New value

    Forgetting Delay Requirements

    1
2
3
4
5
6
7
    // The FRG permit has a 600ms delay
initialize_to_state(MSM_States::ROTATING_FIELD_ENABLED);
EXPECT_FALSE(get_status_flag(MSM_Status_Flags::has_frg_permit_elapsed));

// Must wait for delay
simulate_for(0.65f);
EXPECT_TRUE(get_status_flag(MSM_Status_Flags::has_frg_permit_elapsed));

    Ignoring Silent Parameter

    1
2
3
4
5
6
7
    // Debug output enabled
EXPECT_TRUE(is_in_state(MSM_States::READY_TO_DRIVE, false));

// Silent check for conditional logic
if (is_in_state(MSM_States::READY_TO_DRIVE)) {
    // Do something
}
